# Examples

These examples are synthetic but written as if they were real client engagements run by a deployment strategist using `pstack` in a disciplined delivery motion.

## What These Show

- how discovery turns into a concrete ontology and pipeline plan
- how build artifacts expose real blockers, exception paths, and governance decisions
- how review, security, QA, deployment, training, and retrospectives carry the same facts forward instead of inventing new ones
- how a complete artifact chain reads when it is detailed enough for a real customer program team

## Included Engagements

### `engagements/acme-supply-chain/`

A global industrial manufacturer building a control tower for expedite decisions, supplier delay triage, and inventory-risk management across North America and Western Europe.

### `engagements/northstar-healthcare/`

A regional health system building a patient-flow command center for bed assignment, discharge readiness, transfer escalation, and operational capacity management.

## How To Use The Examples

1. Start with the engagement README to understand the customer situation and delivery scope.
2. Read the artifacts in chain order from `BOOTCAMP-SCOPE.md` through `RETRO-REPORT.md`.
3. Compare the two engagements to see what changes by industry and what stays invariant in the DS operating model.
4. Use these as reference quality bars when authoring new customer-facing artifacts.

Every engagement includes the full artifact set defined in `conductor.json` and is intentionally detailed enough to feel like a real field program rather than a toy sample.
