# Improvement Patterns

## What Usually Deserves Promotion

- Repeated ambiguity in the same template section across engagements.
- Repeated safety or governance confusion at the same program phase.
- Repeated benchmark underperformance tied to the same skill boundary.

## What Usually Does Not

- One-off customer politics.
- Purely stylistic preferences from a single DS.
- Changes that widen tool scope without clear eval benefit.
